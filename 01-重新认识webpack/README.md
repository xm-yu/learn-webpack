### 前端工程化的发展

一开始前端写代码，都是写原生 js css，要自己手动将资源引入 html 中。开发效率很低，我们要非常精细管理每个资源标签引入资源的顺序，浪费时间和精力。
09 年 node 诞生，开始逐渐有了很多解决具体问题的工具 想 babel 可以让我们使用更高级的 es 语法 typescript 。less sass 等样式预处理可以让我们在开发样式的时候引入逻辑运算 嵌套继承等结构化语言特性。
这些工程化工具帮我们抹平了浏览器的兼容的问题，js 语言的一些设局缺陷，我们不需要再关注一些低效的技术细节，将更多的注意力放在业务代码上，用更高效的方式编写复杂 庞大的 web 应用

### webpack 给前端前端工程化带来的意义？

webpack 是一个静态资源打包器
webpack 解决了什么问题？
node 诞生出现很多效率工具 比如 babel ts 帮我们解决 js 的诸多问题，less sass 为了们编写样式带来逻辑运算等语言特性，压缩 处理图片的一些工具。
所以我们需要一个工具来帮接入这一系列的工程化工具，实现统一资源构建。

### webpack 总体的设计

前端需要的不同资源需要不同的处理逻辑，且不同类型的文件之间无法信息共通。
webpack 则忽略资源的不同，将所以资源都看做 module -模块对象，用相同的加载、解析、依赖管理、优化、合并流程实现打包，并借助 loader plugin 两种开发接口将资源差异化处理交给社区去实现，我们都可以自己编写不同 loader plugin 去处理资源。
